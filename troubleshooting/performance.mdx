## Performance tips

- **Use hardware encoding** (NVENC for NVIDIA, VAAPI for AMD/Intel) instead of software x264 encoding. Hardware encoding significantly reduces latency.
- **NVIDIA GPU**: Stick to the default rendering resolution. WiVRn is not fully optimized for NVIDIA hardware, and latency can increase notably at higher-than-default resolutions.
- **Network**: Use Wi-Fi 6 or a wired (USB) connection for the best quality and lowest latency.
- **Codec**: Try different codecs (H.264, H.265, AV1). AV1 may be more efficient on supported hardware and headsets.
- **Resolution**: If you experience quality issues, decrease the rendering resolution in the WiVRn headset app.

## Timing analysis

WiVRn can record detailed timing data to a CSV file for offline analysis:

```bash
WIVRN_DUMP_TIMINGS=/tmp/wivrn-timings.csv wivrn-server
```

## Perfetto profiling

[Perfetto](https://perfetto.dev/) provides detailed system-wide profiling for both the headset and the server.

### HMD profiling

<Steps>
  <Step title="Enable persistent tracing on the headset">
    ```bash
    adb shell setprop persist.traced.enable 1
    ```
  </Step>
  <Step title="Open the Perfetto UI">
    Go to [https://ui.perfetto.dev/](https://ui.perfetto.dev/), connect to your device, and configure tracing as needed.

    See the [Android tracing quickstart](https://perfetto.dev/docs/quickstart/android-tracing) for detailed instructions.
  </Step>
</Steps>

### Server profiling

<Steps>
  <Step title="Build WiVRn with tracing enabled">
    Build with the `XRT_FEATURE_TRACING=ON` CMake option to enable [Monado tracing with Perfetto](https://monado.pages.freedesktop.org/monado/tracing-perfetto.html).
  </Step>
  <Step title="Start the server with tracing enabled">
    ```bash
    XRT_TRACING=true wivrn-server
    ```
  </Step>
</Steps>

## Environment variables reference

| Variable | Description | Default |
|---|---|---|
| `XRT_LOG` | Log level for the XR runtime (`trace`, `debug`, `info`, `warn`, `error`) | `warn` |
| `XRT_COMPOSITOR_LOG` | Compositor log level (`trace`, `debug`, `info`, `warn`, `error`) | `info` |
| `OXR_VIEWPORT_SCALE_PERCENTAGE` | Per-application viewport scale percentage. Set on the application, not the server. | `100` |
| `WIVRN_DUMP_VIDEO` | Path prefix for video frame dumps (e.g., `/tmp/video-dump`) | unset |
| `WIVRN_DUMP_TIMINGS` | Path for timing CSV output (e.g., `/tmp/wivrn-timings.csv`) | unset |
| `WIVRN_LOGLEVEL` | Log level for the native headset client | unset |
| `WIVRN_AUTOCONNECT` | Auto-connect to the first discovered server | unset |
| `XRT_TRACING` | Enable Perfetto tracing (`true`/`false`) | `false` |
| `VK_INSTANCE_LAYERS` | Vulkan validation layers (e.g., `VK_LAYER_KHRONOS_validation`) | unset |
| `XR_RUNTIME_JSON` | Path to the OpenXR runtime configuration file | unset |
